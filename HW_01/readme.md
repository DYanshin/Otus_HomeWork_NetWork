## ЗАДАЧА 2. Создание сети и настройка основных параметров устройства

### Шаг 1. Настройка базовых параметров коммутатора

- В режиме глобальной конфигурации копируем следующие базовые параметры конфигурации и вставляем их в файл на коммутаторе S1:
  
no ip domain-lookup 

hostname S1

service password-encryption

enable secret class

banner motd #

Unauthorized access is strictly prohibited. 

- Назначаем IP-адрес интерфейсу SVI на коммутаторе:
  
S1# configure terminal > interface vlan 1 > ip adress 192.168.1.2 255.255.255.0

![Часть12](https://github.com/user-attachments/assets/84b036ca-54cc-4c1a-8050-51cb777815f2)

- Используем параметр logging synchronous:

S1(config)# line con 0
S1(config-line)# logging synchronous 

- Настраиваем каналы виртуального соединения для удаленного управления (vty), чтобы коммутатор разрешил доступ через Telnet

S1# line vty 0 15 > transport input telnet > login > password <пароль> > login 

Команда login для включения аутентификации при подключении, требуя пароль для доступа к удаленному управлению.

## ЗАДАЧА 3. Проверка сетевых подключений



### Шаг 1. Отображаем конфигурацию коммутатора

- Используем консольное подключение на компьютере PC0 для отображения и проверки конфигурации коммутатора.
Команда S1# show run

- Проверяем параметры VLAN 1:

S1# show interface vlan 1 

Полоса пропускания этого интерфейса - BW 100000 Kbit

### Шаг 2. Протестируем сквозное соединение, отправив эхо-запрос

- В командной строке компьютера PC0 с помощью утилиты ping проверяем связь с адресом PC0:

C:\> ping 192.168.1.10 

![Часть14](https://github.com/user-attachments/assets/bf405ff7-2f42-468a-adb4-d1ed4b33bec3)

- Из командной строки компьютера PC0 отправляем эхо-запрос на административный адрес интерфейса SVI коммутатора S1:

C:\> ping 192.168.1.2

![Часть15](https://github.com/user-attachments/assets/e1c1f2c2-c782-4706-ad6f-2a8837604ad1)

### Шаг 3. Проверяем удаленное управление коммутатором S1

- Открываем Tera Term - программу эмуляции терминала с возможностью Telnet
 
- Выбераем сервер Telnet и указываем адрес управления SVI для подключения к S1.  Пароль: class
  
- После ввода пароля class оказываемся в командной строке пользовательского режима. Для перехода в исполнительский режим EXEC   введите команду enable и используйте секретный пароль class
