# Лабораторная работа. Базовая настройка коммутатора

## Задачи:
 ### 1. Проверка конфигурации коммутатора по умолчанию
 ### 2. Создание сети и настройка основных параметров устройства
          • Настроить базовые параметры коммутатора
          • Настроить IP-адрес для ПК
 ### 3. Проверка сетевых подключений
          •	Отобразить конфигурацию устройства
          •	Протестировать сквозное соединение, отправив эхо-запрос.
          •	Протестировать возможности удаленного управления с помощью Telnet

## Решение:
### Шаг 1. Создаем сеть согласно топологии
   - Подсоединяем консольный кабель (синего цвета) от Swith0, как показано в топологии. На данном этапе не подключаем кабель Ethernet (это и есть витая пара) компьютера PC0:

   ![Часть1](https://github.com/user-attachments/assets/86371f26-6a21-48c0-858c-5833374385e1)

   - Установливаем консольное подключение к коммутатору с компьютера PC0 с помощью программы эмуляции терминала (признак успешного консольного подключения ПК к коммутатору появления Switсh> в терминале):

   ![Часть2](https://github.com/user-attachments/assets/068313fb-59dc-4b7d-9b5f-230e7e72116d)

  ВОПРОС: Почему нужно использовать консольное подключение для первоначальной настройки коммутатора? Почему нельзя подключиться к коммутатору через Telnet или SSH?
  
  ОТВЕТ:  Консольное подключение необходимо для начальной настройки коммутатора, так как устройства «из коробки» не имеют IP-адреса, логина или активных сетевых служб.

  ### Шаг 2. Проверить настройки коммутатора по умолчанию
  - Подключаем доступ к привилегированному режиму командой enable в терминале (через этот режим можно получить прямой доступ к режиму глобальной конфигурации и командам, 
    используемым для настройки рабочих параметров) Появляется Switch#:


  ![Часть3](https://github.com/user-attachments/assets/e74d06a0-efd9-49cd-86ea-7b7a4e87c476)

  - В привилегированном режиме вводим команду show running-config (Switch# show running-config):

  ![Часть4](https://github.com/user-attachments/assets/3681272b-a473-4aac-b959-9000847b5381)

  - Изучаем текущий файл running configuration:
    
       на коммутаторе 2960 имеются 24 интерфейса FastEthernet (0 из 24)
    
       на коммутаторе 2960 имеются 2 интерфейса GigabitEthernet  (0 из 2)

       диапазон значений, отображаемых в vty-линиях:

       команда line vty 0 4 — открывает 5 виртуальных линий (0, 1, 2, 3, 4) для удалённого доступа

       Команда line vty 0 15 — указывает на диапазон значений, которые присваиваются каждой активной сессии — от 0 до 15

 ![Часть5](https://github.com/user-attachments/assets/cdf33388-9a96-41a4-9d82-ae592c3f5b15)

 - Сообщение «Файл загрузочной конфигурации (startup configuration) не найден в энергонезависимом ОЗУ (NVRAM)» появляется, если файл не найден. Это может произойти из-за:

   1. Устройство не имеет настроек, отличных от настроек по умолчанию. В этом случае нужно выполнить очистку настроек и перезагрузить устройство.

   2. Файл не сохранён — если не выполнена команда copy running-config startup-config, то все изменения в конфигурации будут утеряны
